Module dynamixel-bus

Library for Dynamixel protocol.

This library allows to manipulate devices that use Dynamixel protocol, such as AX-12 robotic servo motors. When available, a dynamixel bus will be represented by a Device object in toribio.devices table. The device will be named (as an example), "dynamixel:/dev/ttyUSB0".

has_errors (code)
Decodes dynamixel error codes.

Parameters:

  • code A dynamixel error code as returned by the different motor methods.

Returns:

    A set of error strings. The possible error strings are 'ERROR_INPUT_VOLTAGE', 'ERROR_ANGLE_LIMIT', 'ERROR_OVERHEATING', 'ERROR_RANGE', 'ERROR_CHECKSUM', 'ERROR_OVERLOAD', 'ERROR_INSTRUCTION'.
    The table is double-indexed by entry number to allow array-like traversal.

Usage:

     local errorset = dynamixel.has_errors(0x10+0x08)
print ("has errors:" #errorset>0 )
print ("has range error:", errorset['ERROR_RANGE']==true )

Signals emitted by this device.

Fields:

  • discovery_end The discovery of attached motors has finished.
conf
Configuration Table. When the start is done automatically (trough configuration), this table points to the modules section in the global configuration table loaded from the configuration file.

Fields:

  • load whether toribio should start this module automatically at startup.
  • filename the device file for the serial bus (defaults to '/dev/ttyUSB0').
  • serialtimeout the timeout when waiting for a response from an actuator (defaults to 0.05s)
  • stty_flags parameters to pass to stty for configuring the serial bus.
  • serialspeed serial bus speed in bps (defaults to 1000000).
